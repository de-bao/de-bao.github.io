# 前端动态加载文章功能

## 🎯 实现方案

### 架构改进

**之前**：
- 前端完全依赖 Jekyll 静态生成
- 新文章需要重新构建才能显示
- 无法实时获取数据库中的文章

**现在**：
- **混合模式**：静态文章 + 动态 API 加载
- 页面加载时自动从 API 获取最新文章
- 实时显示数据库中的已发布文章

---

## 🔄 工作流程

```
页面加载
    ↓
显示静态文章（Jekyll 构建时生成）
    ↓
同时从 API 获取已发布文章
    ↓
动态渲染新文章
    ↓
合并显示（去重）
```

---

## ✨ 功能特性

### 1. 自动加载
- 页面加载时自动从 API 获取文章
- 无需手动刷新
- 实时显示最新内容

### 2. 智能去重
- 自动检测重复文章（通过 slug）
- 静态文章优先显示
- 动态文章补充显示

### 3. 无缝集成
- 动态文章使用相同的样式
- 支持所有筛选功能（标签、年份、搜索）
- 支持文章详情页

### 4. 降级处理
- API 失败不影响静态文章显示
- 保持网站可用性

---

## 📝 实现细节

### 文章列表页 (`_layouts/blog.html`)

1. **静态文章**：Jekyll 构建时生成，作为后备
2. **动态加载**：页面加载时调用 API
3. **合并显示**：去重后统一显示
4. **筛选支持**：同时支持静态和动态文章

### 文章详情页

1. **静态文章**：使用 Jekyll 路由（`/技术/文章标题/`）
2. **动态文章**：通过 JavaScript 动态加载内容
3. **URL 管理**：使用 History API 更新 URL

---

## 🔧 API 集成

### 使用的 API 端点

- `GET /api/posts?status=published` - 获取已发布文章列表
- `GET /api/posts/:slug` - 获取单篇文章详情

### 配置

API 地址在 `_layouts/blog.html` 中配置：
```javascript
const API_BASE = 'https://api.debao.qzz.io/api';
```

## 🔄 与 GitHub 同步的关系

### 两种方案并存

1. **GitHub 同步**（推荐用于长期存储）
   - 发布时自动提交到 GitHub
   - 文章永久保存在 Git 仓库
   - 下次构建时成为静态文章

2. **API 动态加载**（实时显示）
   - 立即显示新发布的文章
   - 无需等待构建
   - 实时更新

### 最佳实践

- **新文章**：通过 API 立即显示
- **已同步文章**：下次构建后成为静态文章
- **混合模式**：静态 + 动态，最佳体验

## 🐛 故障排查

### 问题：动态文章不显示

**检查**：
1. API 是否正常：`curl https://api.debao.qzz.io/api/posts`
2. 浏览器控制台是否有错误
3. 网络请求是否成功

### 问题：文章重复显示

**解决**：
- 系统会自动去重
- 如果仍有问题，检查 slug 是否一致

### 问题：文章详情无法加载

**解决**：
- 检查 API 是否返回正确数据
- 检查 slug 是否正确
- 降级到静态页面
