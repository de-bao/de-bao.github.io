# 🔐 GitHub 认证配置指南

## 📌 当前问题

推送时需要输入 GitHub 的用户名和密码，但系统无法读取密码。

## 🎯 解决方案

### 方案一：使用 Personal Access Token（推荐）

GitHub 从 2021 年 8 月起不再支持使用账户密码进行 Git 操作，必须使用 Personal Access Token。

#### 步骤 1：创建 Personal Access Token

1. **登录 GitHub**
   - 访问：https://github.com
   - 使用你的账号登录

2. **进入设置页面**
   - 点击右上角头像
   - 选择 **Settings**（设置）

3. **找到 Developer settings**
   - 滚动到页面最底部
   - 点击左侧菜单的 **Developer settings**

4. **创建新的 Token**
   - 点击 **Personal access tokens**
   - 选择 **Tokens (classic)**
   - 点击 **Generate new token** → **Generate new token (classic)**

5. **配置 Token**
   - **Note（备注）：** 输入一个描述，例如 "Baud 博客推送"
   - **Expiration（过期时间）：** 选择过期时间（建议选择 90 天或更长时间）
   - **Select scopes（选择权限）：** 勾选 `repo`（完整仓库访问权限）
   - 点击 **Generate token**（生成令牌）

6. **复制 Token**
   - ⚠️ **重要：** Token 只显示一次，请立即复制保存！
   - 如果丢失，需要重新生成

#### 步骤 2：使用 Token 推送

**方法 A：在推送时输入**

```bash
cd /home/10355407/Baud
git push origin main
```

当提示输入：
- **Username（用户名）：** 输入 `de-bao`
- **Password（密码）：** 输入刚才复制的 **Personal Access Token**（不是 GitHub 密码）

**方法 B：配置 Git Credential Helper（更便捷）**

```bash
# 配置 Git 记住凭据
git config --global credential.helper store

# 然后推送（输入一次用户名和 Token）
git push origin main
```

之后 Git 会记住你的凭据，下次推送就不需要输入了。

### 方案二：使用 SSH 密钥（更安全，推荐长期使用）

#### 步骤 1：检查是否已有 SSH 密钥

```bash
ls -al ~/.ssh
```

如果看到 `id_rsa` 或 `id_ed25519` 文件，说明已有密钥。

#### 步骤 2：生成 SSH 密钥（如果没有）

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
```

- 按回车使用默认文件位置
- 可以设置密码（可选，直接回车跳过）

#### 步骤 3：复制公钥

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出的全部内容。

#### 步骤 4：添加到 GitHub

1. 访问：https://github.com/settings/keys
2. 点击 **New SSH key**
3. **Title：** 输入描述（例如 "我的电脑"）
4. **Key：** 粘贴刚才复制的公钥
5. 点击 **Add SSH key**

#### 步骤 5：更改远程仓库地址为 SSH

```bash
cd /home/10355407/Baud
git remote set-url origin git@github.com:de-bao/de-bao.github.io.git
```

#### 步骤 6：测试连接

```bash
ssh -T git@github.com
```

应该看到：`Hi de-bao! You've successfully authenticated...`

#### 步骤 7：推送

```bash
git push origin main
```

### 方案三：使用 GitHub CLI（最简单）

#### 安装 GitHub CLI

```bash
# Ubuntu/Debian
sudo apt install gh

# 或使用其他安装方式
```

#### 登录

```bash
gh auth login
```

按照提示操作，选择：
- GitHub.com
- HTTPS
- 浏览器登录（推荐）或 Token

#### 推送

```bash
cd /home/10355407/Baud
git push origin main
```

## 🚀 快速开始（推荐流程）

### 如果你想要最简单的方式：

1. **创建 Personal Access Token**（按上面的步骤 1）
2. **配置 Git 记住凭据：**
   ```bash
   git config --global credential.helper store
   ```
3. **推送一次：**
   ```bash
   cd /home/10355407/Baud
   git push origin main
   ```
   - 用户名：`de-bao`
   - 密码：粘贴你的 Personal Access Token
4. **以后推送就不需要输入了！**

## 💡 推荐做法

- **短期使用：** Personal Access Token + Credential Helper
- **长期使用：** SSH 密钥（更安全，一次配置永久使用）

## 🆘 需要帮助？

如果遇到问题：
1. 检查 Token 是否正确复制（没有多余空格）
2. 确认 Token 有 `repo` 权限
3. 检查 Token 是否过期
4. 查看 GitHub 的错误提示信息
