## 🔧 方法二：手动命令（分步详解）

### 第 1 步：进入项目目录

```bash
cd /home/10355407/Baud
```

### 第 2 步：检查当前状态

```bash
git status
```

**应该看到类似这样的输出：**
```
位于分支 main
您的分支领先 'origin/main' 共 X 个提交。
无文件要提交，干净的工作区
```

### 第 3 步：添加所有更改（如果有新文件）

```bash
git add .
```

**说明：** 这会把所有新文件和修改添加到暂存区

### 第 4 步：提交更改（如果有新更改）

```bash
git commit -m "更新博客功能和样式"
```

**说明：** 
- `-m` 后面是提交信息，可以改成任何描述性文字
- 例如：`"添加新文章"`、`"更新样式"`、`"修复bug"` 等

### 第 5 步：推送到 GitHub

```bash
git push origin main
```

**说明：**
- `origin` 是远程仓库的别名
- `main` 是分支名称
- 这会把本地的提交推送到 GitHub

### 第 6 步：输入 GitHub 凭据（如果需要）

如果提示输入用户名和密码：
- **用户名：** 你的 GitHub 用户名（de-bao）
- **密码：** 需要使用 **Personal Access Token**（不是 GitHub 密码）

**如何获取 Personal Access Token：**
1. 登录 GitHub
2. 点击右上角头像 → Settings
3. 左侧菜单找到 Developer settings
4. Personal access tokens → Tokens (classic)
5. Generate new token (classic)
6. 勾选 `repo` 权限
7. 生成并复制 token（只显示一次，要保存好）

## 🔍 常见问题解决

### 问题 1：提示 "Permission denied"

**原因：** 没有权限访问仓库

**解决方法：**
1. 检查是否已登录 GitHub
2. 使用 Personal Access Token 而不是密码
3. 确认仓库所有权或访问权限

### 问题 2：提示 "remote: Invalid username or password"

**原因：** 认证失败

**解决方法：**
1. 使用 Personal Access Token
2. 或者配置 SSH 密钥（更安全）

### 问题 3：提示 "Updates were rejected"

**原因：** 远程仓库有本地没有的更新

**解决方法：**
```bash
# 先拉取远程更新
git pull origin main

# 解决可能的冲突后，再推送
git push origin main
```

### 问题 4：提示 "fatal: not a git repository"

**原因：** 不在 Git 仓库目录中

**解决方法：**
```bash
cd /home/10355407/Baud
```

### 问题 5：网站没有更新

**原因：** GitHub Pages 还在构建中

**解决方法：**
1. 等待 1-5 分钟
2. 清除浏览器缓存（Ctrl+F5 或 Cmd+Shift+R）
3. 检查 GitHub 仓库的 Settings → Pages 查看部署状态
